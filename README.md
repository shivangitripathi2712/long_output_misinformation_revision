# Long-Output Misinformation Revision using Evidence-Grounded Sequential Editing

This repository contains the implementation of a paragraph-level misinformation revision framework for long-form outputs generated by Large Language Models (LLMs). The project focuses on detecting and correcting misinformation in long passages by using evidence retrieval, agreement checking, and sequential sentence-level revision.

The goal of this work is to revise long LLM-generated responses so that unsupported or factually incorrect statements are corrected while preserving the original meaning, structure, and fluency as much as possible.

---

## Project Overview

Large Language Models often generate long responses that are fluent but may contain unsupported, fabricated, or factually incorrect information. These errors are difficult to correct because misinformation may appear in only a few sentences while the rest of the paragraph may already be correct.

This project addresses that problem through an evidence-grounded sequential revision pipeline.

Instead of rewriting the whole paragraph at once, the framework processes the long output in smaller claim-like units. Each sentence or claim is checked against retrieved evidence, and only the unsupported or contradictory parts are revised. This helps preserve the original response while improving factual consistency.

---

## Research Motivation

Most hallucination correction methods focus on short claims or single-sentence outputs. However, real LLM outputs are often long paragraphs containing multiple factual statements. A single paragraph may include both correct and incorrect information.

This creates three major challenges:

1. The model should not rewrite the entire paragraph unnecessarily.
2. The model should correct only the unsupported or misleading parts.
3. The final revised paragraph should remain coherent and close to the original output.

This repository implements a framework designed to support paragraph-level misinformation correction using evidence-based revision.

---

## Main Contribution

This project extends the idea of retrieve-and-revise factual correction to long-form LLM outputs.

The main contributions of this implementation are:

- Sentence-level decomposition of long LLM-generated text
- Question generation for each claim or sentence
- Evidence retrieval from external sources
- Agreement gating to decide whether revision is needed
- Sequential sentence-level revision
- Evidence selection for attribution and analysis
- Exporting outputs for qualitative and quantitative evaluation

---

## Framework Pipeline

The framework follows these major steps:

### 1. Input Long Output

The system takes a long LLM-generated passage or claim as input. This passage may contain misinformation, unsupported statements, or hallucinated facts.

### 2. Sentence or Claim Decomposition

The long passage is divided into smaller units such as sentences or atomic claims. This allows the framework to analyze each part independently instead of rewriting the entire paragraph.

### 3. Question Generation

For each sentence or claim, the system generates fact-checking questions. These questions are used to search for evidence related to the claim.

### 4. Evidence Retrieval

The generated questions are used to retrieve relevant evidence passages. The evidence is then used to verify whether the claim is supported, contradicted, or not clearly grounded.

### 5. Agreement Gate

The agreement gate decides whether the retrieved evidence agrees with the current claim.

- If the evidence supports the claim, the claim is kept mostly unchanged.
- If the evidence contradicts the claim, the claim is sent for revision.
- If the evidence is unclear or weak, the framework avoids unnecessary aggressive rewriting.

### 6. Sequential Revision

The framework revises the passage sentence by sentence. This sequential process helps maintain paragraph-level coherence and reduces unnecessary edits.

### 7. Evidence Selection

The framework selects useful evidence snippets that can explain or justify the final revised output.

### 8. Final Revised Output

The final output is a corrected version of the original long passage. The goal is to improve factual accuracy while preserving the original style and meaning.

```bash

cd long_output_misinformation_revision
```



## Input File Format

The input should be a JSONL file where each line contains one example.

A simple format is:

```json
{"input_info": {"claim": "The generated long-form answer or paragraph goes here."}}
```

Example file:

```text
statement.jsonl
```

Example content:

```json
{"input_info": {"claim": "The Eiffel Tower is located in Berlin and was built in 1900."}}
{"input_info": {"claim": "Marie Curie discovered radium and polonium and won Nobel Prizes in two scientific fields."}}
```

The field name can be changed using the `--claim_field` argument.

---

## How to Run the Full Sequential Revision Pipeline

Run the main pipeline using:

```bash
python run_editor_sequential.py \
  --input_file statement.jsonl \
  --output_file output.jsonl \
  --model_name text-davinci-003 \
  --claim_field claim
```

Explanation of arguments:

```text
--input_file      Path to the input JSONL file
--output_file     Path where the revised output will be saved
--model_name      Name of the language model used for revision
--claim_field     Field name containing the claim or passage
```

If your code uses a newer model, replace the model name. For example:

```bash
python run_editor_sequential.py \
  --input_file statement.jsonl \
  --output_file output.jsonl \
  --model_name gpt-3.5-turbo \
  --claim_field claim
```

## Important Notes

This framework is designed for research on misinformation correction and hallucination mitigation.

The evidence retrieval step is important. If the retrieved evidence is weak or irrelevant, the revision quality may also be affected. Therefore, the quality of evidence directly influences the final corrected output.

The framework should avoid unnecessary rewriting when the original claim is already supported by evidence. This is important because the goal is not to paraphrase the entire paragraph, but to correct misinformation while preserving the original response as much as possible.
